群發訊息

整合模板、名單與參數的群發訊息功能設計

關於作品

題目 - 群發訊息

在 CDP 系統中，行銷人員經常需要向大量會員發送 EDM 或 SMS，例如生日優惠券或活動通知。
而這不僅是單純的訊息發送，更是涉及到個人參數與模板的使用，並且要知道開封率的成效結果。

因此本功能的目標是設計一套 支援個人化參數與多狀態管理的群發訊息系統，協助行銷人員安全且有效地完成訊息發送。

執行時間:

2 weeks

扮演角色:

UX: User Flow
UI: Wireframe、UI Flow

工作流程

需求接收:

PM 提供初步需求心智圖，包含訊息群發相關模組：

    主要包含:
  • 訊息發送
  • 名單管理
  • 模板管理
  • 發送流程
  • 發送紀錄

設計階段需要將這些功能整理為清晰的系統流程與使用者操作路徑。

設計重點:

本功能設計重點在於 降低群發訊息錯誤率，並提升操作效率。

1.參數驗證機制
訊息模板可插入個人化參數，例如：{name}、{coupon}

    系統在發送前會檢查：
  • 模板參數
  • 名單欄位

若欄位不匹配，系統會顯示警告，避免寄出錯誤訊息。

2.發送前預覽

    提供預覽頁，讓使用者確認：
  • 訊息內容
  • 發送名單
  • 發送時間

降低誤發送風險。

3.多狀態管理

訊息發送包含多個狀態，除了一般建立狀態以外還加入了審核狀態

    提供預覽頁，讓使用者確認：
  • 草稿
  • 審核中
  • 未通過
  • 等待中
  • 處理中
  • 已發送
  • 發送失敗

在列表上透過狀態標示，讓行銷人員能快速掌握每則訊息的發送進度。

UI Flow規劃:

在確認整體使用流程後，我進一步規劃 UI Flow，將每個設定步驟轉換為具體的介面操作，使使用者能依據名單參數與訊息模板，逐步完成群發訊息的設定。

功能說明

1.上傳參數

行銷人員必須先上傳準備好的參數名單，除了基本的{email}、{name}，也要輸入所要使用的參數值，最常見的就是{coupon}

2.群發列表

展示每筆建立的群發訊息狀態

3.編輯群發訊息

建立群發名稱並選擇設定名單與模板，右下能及時查看訊息服務商的可用餘額，當不足時始行銷人員可以立刻儲值

4.內容預覽

預覽設定內容與名單是否正確，也能事先發送測試，寄送前檢查匹配名單與模板的參數名稱與值是否為空。

5.成效檢視

透過多種發送狀態與成效頁面，讓使用者能清楚掌握訊息處理進度與寄送結果，提升整體操作效率與可控性。

功能價值

本設計整合名單管理、模板設定與個人化參數，使行銷人員能快速建立群發訊息。
透過清晰的 UI Flow 與發送前驗證機制，降低模板參數與名單資料不匹配所造成的發送錯誤。
同時透過多狀態管理與成效頁面，讓使用者能即時掌握訊息處理進度與發送結果。
