群發訊息

讓行銷人員快速發送「客製化大量訊息」，提升開封率並降低發送成本

支援 EDM / SMS｜CSV 名單上傳｜發送成效追蹤

群發訊息｜讓行銷人員能大量發送「客製化內容」，並掌握發送成效

我的角色:

整合模板系統與名單機制，設計可規模化的訊息發送流程

專案概述

  • 專案類型: Saas CDP
  • 使用者: 行銷人員
  • 功能類型: 既有功能整合與流程優化

商業目標:

  1. 提升大量發送的效率（降低人工操作時間）
  2. 提升訊息開封率（透過個人化內容）
  3. 降低發送錯誤與成本浪費

成果:

  1. 建立「模板 × 名單參數」的群發機制
  2. 支援客製化訊息內容（如姓名、優惠資訊）
  3. 提供發送成效追蹤（成功 / 失敗 / 開封）
  4. 優化發送流程，降低錯誤率

關鍵決策:

  1. 使用門檻問題，採用 CSV 提升可用性
  2. 發送風險問題，以導入預覽及審核機制

問題定義

現有系統存在以下限制：

  1. 僅能單封發送或發送相同內容
  2. 無法批次處理大量名單
  3. 缺乏發送結果與成效追蹤
  4. 無法支援個人化內容

核心問題:

無法同時做到「大量發送」與「個人化內容」，且缺乏成效回饋機制

關鍵流程

針對PM提供的心智圖我將需求直觀拆解為：

名單準備 → 模板設定 → 內容匹配 → 發送設定 → 成效追蹤

轉化為使用者操作流程：

設定模板 → 上傳名單 → 驗證匹配 → 預覽內容 → 發送 → 成效查看

此圖由 PM 提供，作為條件判斷的邏輯基礎

設計拆解

  1. 1.模板系統

    • 支援 EDM / SMS 模板 (現有功能)

  2. 2.名單機制

    • 支援 CSV 批次上傳
    • 定義欄位與參數對應關係

  3. 3.發送與成效

    • 發送前模板與名單匹配
    • 發送前預覽個人化內容
    • 提供發送結果與開封名單

關鍵設計

參數匹配與錯誤防呆:

  • 模板與名單欄位不一致時即時提示
  • 提供替代文字避免內容顯示異常

發送前預覽機制:

  • 顯示實際發送內容（含個人化資訊）

發送審核流程:

  • 發送前需主管審核

解決問題:

  1. 降低錯誤發送風險，確保訊息完整性
  2. 避免錯誤內容被大量發送
  3. 控管發送風險與成本

關鍵決策

在產品策略上，PM 決定採用 CSV 作為名單輸入方式，以降低行銷人員的使用門檻並支援外部資料來源。

因此我的設計重點將著重再於，使用者如何正確的使用格式，降低發送錯誤。

為何使用 csv 檔案上傳?

  • 行銷人員較為熟悉的格式，直接 Excel 操作
  • 不須使用額外串接技術
  • 支援非會員的寄送

預覽與審核機制:

參考市場上既有工具，多數在發送前皆提供預覽機制，其核心目的在於「降低錯誤發送所帶來的成本風險」。

在此脈絡下，產品已規劃預覽與審核流程，我的設計重點在於強化「多層驗證機制」，避免錯誤被放大。

畫面與操作

1.建立模板與匹配名單參數

重點:上傳名單後可再次檢視，降低發送錯誤率

2.新增群發項目

重點:列表清楚檢視各個群發活動的發送狀態

3.設定與預覽

確保發送前可確認個人化內容

4.發送成效

快速辨識發送成功與開封狀態

成效

  1. 將「單筆發送」轉成「批次且客製化發送」
  2. 減少人工操作發送的成本
  3. 降低錯誤發送的成本消耗

功能價值

  1. 補足自動化行銷無法處理的外部名單情境, 與既有行銷工具形成互補相輔相成
  2. 在操作效率與風險控管之間取得平衡
  3. 整合模板、名單與審核流程的複雜情境
