週期標籤

週期分群，實現精準商品推薦

關於作品

題目 - 週期標籤 / 寶寶標籤

與客戶訪談中，得知母嬰商品的購買需求與寶寶成長階段高度相關，不同月份需要不同產品，如奶瓶、副食品與學習餐具。

因此行銷團隊希望能在合適的時間推薦相應商品。為此，我們將需求轉化為產品功能，設計一套週期標籤，依據寶寶月份貼上不同標籤。

行銷人員可透過這些標籤快速建立精準受眾，並在自動化行銷旅程中觸發商品推播，讓品牌在寶寶成長關鍵階段與家長建立密切溝通。

執行時間:

1 month

扮演角色:

UX: User Flow
UI: Wireframe、精稿設計

工作流程

需求理解:

在需求初期，PM 會提供一份功能心智圖，說明客戶需求、資料來源以及預期的操作流程。我會先與 PM 進行需求討論，了解功能的使用情境、資料邏輯與系統限制，確保對功能目標與需求範圍有一致的理解。

流程梳理:

在理解需求後，我將心智圖中的概念重新整理為 User Flow，釐清建立週期標籤時的主要設定步驟，包含資料來源選擇、週期條件設定與後續標籤運用。

應用場景:

為了幫助團隊理解週期標籤的應用方式，我整理了一個母嬰商品的情境示意。
不同商品對應不同寶寶月齡，行銷策略是在寶寶進入該階段前一週推薦商品。

情境:
嬰兒用品店，商家希望能在寶寶各個時期前一週開始推廣不同產品給媽媽會員，所以希望能先幫會員貼上標籤

    產品資訊:
  • 產品A / 適合年齡：0-1 month
  • 產品B / 適合年齡：1-3 month
  • 產品C / 適合年齡：3-6 month
  • 產品D / 適合年齡：6 month 以上

UI Flow & Wireframe 規劃:

在確認使用流程後，我進一步規劃 UI Flow，將每個設定步驟轉換為具體的介面操作，讓使用者可以依序完成資料來源選擇、週期設定與標籤建立。

透過將需求心智圖轉換為 User Flow 與 UI Flow，將抽象的產品需求整理為清晰的操作流程，確保功能設計能順利落地為實際的系統介面。

功能說明

1.資料來源設定

週期標籤計算: 需要依據會員資料中的時間欄位作為基準。
系統支援以下兩種資料來源：
公版 CRM 欄位- (1)生日 (2)註冊日 (3)會員到期日 (4)點數到期日
自訂 CRM 欄位- 僅顯示欄位型態為「時間」的自訂欄位
行銷人員可依需求選擇適合的時間欄位作為週期計算基準。

2.週期標籤設定

使用者可以建立最多 10 個週期標籤。
每個標籤需要設定以下內容：

  • (1)基本資訊
    1. 標籤名稱
    2. 資料欄位（僅顯示時間型欄位)
  • (2)貼標規則
    1. 是否允許重複貼標（不限年度）
    2. 貼標時機: 時間點 之前 / 之後 / 當日
  • (3)標籤生命週期
    1. 是否啟用「標籤自動脫落」
    2. 脫落時間設定（1–180 天）

完成設定後，系統會依據規則開始計算並貼上對應標籤。

3.標籤運算

當標籤建立完成後，系統會依據設定的時間欄位與週期條件，自動計算符合條件的會員並貼上標籤。若設定自動脫落，標籤也會在指定天數後自動移除。

設計考量

  • 1.時間規則需要讓使用者自行設定

    週期標籤的貼標與脫落時間，並沒有固定的計算方式，而是需要依照不同品牌的商品週期或行銷策略進行設定。
    讓行銷人員可以依照不同情境建立週期規則。

  • 2.限制標籤數量，避免過於複雜

    如果週期標籤數量過多，可能會讓設定與管理變得複雜，因此產品設計上限制每週期活動最多只能設定 10 個標籤。

  • 3.確保標籤能與自動化旅程順利搭配

    週期標籤的主要目的是提供後續的行銷應用，因此在設計時需要考慮標籤應用與期標籤名稱異動等問題。

功能價值

週期標籤上線後，行銷人員可以依據 CRM 的時間資料建立不同週期受眾，並直接應用於自動化行銷旅程。
這讓品牌能在會員的重要時間節點進行更精準的溝通，例如：生日優惠、點數到期提醒、會員續約通知、商品週期推薦......
透過週期標籤與自動化旅程的結合，降低行銷人員手動操作成本，並提升行銷活動的精準度。
